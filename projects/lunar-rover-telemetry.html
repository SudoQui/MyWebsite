Lunar Rover Telemetry Subsystem

Telemetry pipeline for rover subsystems, enabling sensor data capture, relay, and monitoring during field-style prototype tests.

Problem or Motivation

Rover prototypes require reliable telemetry for decision making and validation. Existing setups had limited observability and inconsistent data quality during integration testing.

What I Built

  • Sensor ingestion pipeline for motion, power, and environmental measurements.
  • Telemetry packet formatting and transfer to monitoring endpoints.
  • Test harness for validating data integrity under simulated mission conditions.

Tools and Technologies Used

  • Raspberry Pi, embedded sensor interfaces, serial communication
  • Python data acquisition scripts and logging modules
  • Fusion 360 for mechanical fit and enclosure adjustments

My Role and Contributions

  • Developed telemetry acquisition logic and integration scripts.
  • Supported mechanical-electrical integration for sensor routing.
  • Documented test procedures and results for system review sessions.

Challenges and Lessons Learned

  • Noise and packet loss required robust buffering and retry strategies.
  • Mechanical packaging constraints affected cable reliability.
  • Cross-discipline collaboration improved system-level design quality.
